## 🔧 Разработка

### Требования
- Python 3.8+
- python-telegram-bot 20.7+
- SQLite3

### Настройка окружения
```bash
# Клонирование
git clone https://github.com/your-username/forymin.git
cd forymin

# Создание виртуального окружения
python -m venv venv
source venv/bin/activate  # Linux/macOS
# или
venv\Scripts\activate     # Windows

# Установка зависимостей
pip install -r requirements.txt

# Установка зависимостей для разработки
pip install -r requirements-dev.txt  # если есть
```

### Запуск тестов
```bash
# Запуск всех тестов
python test_bot.py

# Запуск конкретного теста
python -m unittest test_bot.TestDatabaseManager
```

### Проверка кода
```bash
# Проверка синтаксиса
python -m py_compile *.py

# Проверка стиля (если установлен flake8)
flake8 *.py

# Проверка типов (если установлен mypy)
mypy *.py
```

## 📝 Стандарты кода

### Python
- Следуйте PEP 8
- Используйте docstrings для функций и классов
- Добавляйте типы (type hints) где возможно
- Пишите понятные имена переменных и функций

### Комментарии
- Комментируйте сложную логику
- Используйте русский язык для комментариев
- Объясняйте "почему", а не "что"

### Документация
- Обновляйте README.md при добавлении новых функций
- Добавляйте примеры использования
- Обновляйте справку по командам

## 🧪 Тестирование

### Написание тестов
- Каждая новая функция должна иметь тесты
- Тесты должны покрывать основные сценарии
- Используйте mock объекты для внешних зависимостей

### Пример теста
```python
def test_new_function(self):
    """Тест новой функции"""
    # Arrange
    input_data = "test"
    
    # Act
    result = new_function(input_data)
    
    # Assert
    self.assertEqual(result, "expected")
```
